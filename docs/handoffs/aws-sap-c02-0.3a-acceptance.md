# AWS SAP-C02 0.3A Acceptance

Status: Accepted
Decision date: 2026-07-18
Accepted by: explicit repository-user instruction

## Decision

Version 0.3A is accepted and closed as a manual research and architecture milestone. The accepted artifacts are sufficient to design and later execute a separately authorized, manually reviewed 0.3B content pilot. Acceptance does not claim exact live-exam simulation and does not authorize final content, tool, pack, MCP, AWS-account, or release implementation in this task.

## Accepted 0.3A decisions

1. The current target is **AWS Certified Solutions Architect - Professional**, exam code `SAP-C02`.
2. The official-source baseline and exam identity are accepted as of the documented 2026-07-18 research date.
3. The [assessment blueprint](../aws/sap-c02-assessment-blueprint.md) version `0.3A.1` is accepted with **overall medium confidence**: high structural confidence and bounded medium style/difficulty confidence.
4. Official AWS exam-guide and sample-question expression remains **analysis-only** unless separate reuse rights are established. This is a fail-closed project-policy decision, not formal legal advice.
5. Original project questions may match permissible assessment grammar but must not reproduce protected, recalled, leaked, commercial, or suspicious question expression.
6. The [complete high-level learning architecture](../aws/sap-c02-learning-architecture.md) version `0.3A.1`, including all 20 official task statements plus two cross-cutting foundations, is accepted as the structural baseline.
7. Progressive realization remains the governing construction model. Partial content must map to the complete architecture, resolve prerequisites visibly, and never imply full certification coverage.
8. Domain 1, task 1.4 / `SAP-ORG-04`, **design a multi-account AWS environment**, is accepted as the 0.3B pilot.
9. The first pilot will use public AWS documentation and normal public source retrieval only.
10. MCP configuration and AWS-account access are deferred. The pilot requires no AWS credential, private documentation, authenticated Skill Builder material, lab, or AWS resource.

## Fixed 0.3B content boundary

- Domain 1 only, task 1.4 / `SAP-ORG-04`.
- Two original, cited lessons.
- Approximately 24–30 source-bound claims; the final number follows atomicity and review needs rather than quota filling.
- Exactly five original scenario questions: three single-response and two multiple-response.
- No final lesson or question text exists at this acceptance point.
- Text-only by default. An already-supported static PNG may be considered only if a later design record demonstrates material need and completes the existing format-0.3 source/rights/accessibility/non-leakage approval path.
- No AWS account, lab, private source, authenticated Skill Builder material, or MCP installation.
- The realization completes only the declared task slice and bridges; it does not complete Domain 1 or SAP-C02 preparation.

## Accepted evidence and rights boundary

The accepted evidence baseline is the [source register](../aws/sap-c02-source-register.md), [assessment research](../aws/sap-c02-assessment-research.md), and [rights/reuse analysis](../aws/sap-c02-rights-and-reuse.md). Authority, rights, and confidence remain separate. Public availability is not permission to redistribute question expression.

The governing originality rule is: **copy the assessment grammar, not the assessment sentences**. Similarity review uses the abstract blueprint and provenance of independent construction; unsafe question corpora are not retained merely for comparison.

## Accepted curriculum boundary

The accepted pilot realization includes bridge-sized treatment/checks for `SAP-FND-01`, `SAP-FND-02`, and the relevant `SAP-ORG-02` distinction among identity permissions, preventive guardrails, federation, audit, and delegated administration. `SAP-ORG-01` and `SAP-ORG-05` remain recommended context unless a selected scenario makes them material.

## Approval consequence

This acceptance authorizes the repository to treat 0.3A as closed and to design the exact 0.3B authoring boundary. It does not cause draft sources, claims, lessons, questions, approvals, packs, or capabilities to exist. A later explicitly invoked 0.3B content task must follow the accepted scope, the four Proposed ADRs applicable at that time, and the human-review gates in the final design handoff.

## Supersession and freshness

An official exam-code replacement, guide/domain/task change, source-rights change, or material source conflict triggers impact review before 0.3B claim/question approval. It does not silently rewrite this historical acceptance; a new accepted baseline must supersede it explicitly.
