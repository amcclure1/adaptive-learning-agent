# Source and Claim Independent-Verification Protocol

Status: protocol version `ala-independent-ai-verification-1`

1. Start a fresh verifier invocation and record available model/provider/configuration metadata.
2. Load the exact project, accepted blueprint/architecture/realization, target references, and passing structural report. Do not load hidden author reasoning.
3. Independently open every source's canonical public URL. Verify identity, authority, currency, rights treatment, access limits, locator stability, and whether a more primary/current source exists.
4. Review every claim's full statement—not keywords—against its exact cited proposition and locator. Search official documentation for exceptions, limitations, recent changes, and conflicting guidance.
5. Check category, atomicity, applicability, Region/account/configuration/time sensitivity, freshness horizon, objective scope, and wording independence.
6. For recommendations, verify every exact premise digest and decide whether the premises actually entail the recommendation under its stated criterion. A citation on the recommendation cannot replace an atomic premise.
7. Create one immutable finding per independently actionable defect. Cite an official source and precise locator for each material finding. Never silently edit the target.
8. Give every source and claim exactly one disposition and finalize counts. `verified` requires no blocking finding; a low/informational note must be explicitly nonblocking.
9. After revision, use a fresh invocation. Supply prior findings as explicit resolution checks, then repeat the complete review over changed artifacts.

Prohibited inputs include dumps, recalled live questions, leaked keys, unauthorized banks, private employer/customer material, and model memory as current-service evidence. Sampling is forbidden unless an experiment explicitly defines it.
