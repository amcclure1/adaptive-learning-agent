# Version 0.2.1-alpha.1 – Static Asset Pilot

Status: **PRERELEASE**

Release date: 2026-07-19

`v0.2.1-alpha.1` is the approved format-0.3 static-asset pilot. It adds one bounded official diagram workflow while preserving the runtime-independent deterministic core and all earlier pack behavior.

The Python distribution version is the PEP 440 equivalent `0.2.1a1`.

## Added

- Strict pack format 0.3.
- Declared static local PNG assets.
- Standard-library PNG structure, framing, chunk-order, dimension, and CRC validation.
- SHA-256 integrity over exact raw asset bytes and format-0.3 pack-digest coverage of declared paths and raw bytes.
- Confined normalized asset paths, closed inventory validation, duplicate/alias rejection, and bounded asset count, size, and dimensions.
- Ordered question and lesson asset references.
- Core-issued `ala-pack-asset-v1` logical references bound to installed pack identity, version, digest, and asset ID.
- Additive asset capabilities, summaries, and descriptors under the existing runtime-neutral contract.
- The exact official NCVEC E7B10–E7B12 pilot with one shared official Figure E7-1 PNG.
- Digest-covered caption, alternative text, and terminal fallback support.
- Deterministic pre-answer leakage lint for explicit answer markers and complete keyed-option text.
- Explicit human approval of asset identity, source mapping, visual fidelity, redistribution policy, caption, alternative text, terminal fallback, mappings/order, rights, accessibility, and non-leakage across all three questions.

## Preserved

- Pack formats 0.1 and 0.2 and their golden digests.
- SQLite schema version 1.
- Deterministic scoring and question selection.
- Session, presentation, and attempt behavior.
- Retry safety and immutable-attempt conflict handling.
- Challenge quarantine.
- The exact ten-operation catalog.
- Runtime-neutral contract version 0.1.
- Offline pack validation, installation, and study.
- A runtime-independent core with no Hermes, MCP, provider, or network dependency.

## Installation and use

Python 3.12 or newer and Git are required. Hermes v0.18.2 is required only for the verified conversational path.

```powershell
git clone --branch v0.2.1-alpha.1 https://github.com/amcclure1/adaptive-learning-agent.git
Set-Location adaptive-learning-agent
python -m venv .venv
& .\.venv\Scripts\Activate.ps1
python -m pip install .
```

Run the standard-library test suite:

```powershell
$env:PYTHONPATH='src'
python -m unittest discover -s tests -v
```

For the verified Hermes workflow, use the existing isolated profile from a trusted checkout:

```powershell
$env:HERMES_ENABLE_PROJECT_PLUGINS='1'
& "$env:USERPROFILE\.local\bin\hermes.exe" -p adaptive-learning-dev --skills adaptive-learning:adaptive-learning
Remove-Item Env:HERMES_ENABLE_PROJECT_PLUGINS
```

Project-plugin discovery is trusted-checkout behavior. The pack remains usable through deterministic Python tools and the approved terminal fallback without native image presentation.
